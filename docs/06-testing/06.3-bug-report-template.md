# Plantilla de Bug Report — GuideFlow

## Plantilla

```markdown
### Título del Bug
[Descripción breve en una línea]

### Severidad
- [ ] 🔴 Crítico (no se puede usar la app)
- [ ] 🟡 Mayor (funcionalidad rota pero hay workaround)
- [ ] 🟢 Menor (cosmético o edge case)

### Pasos para reproducir
1. Ir a...
2. Hacer click en...
3. Rellenar...

### Resultado esperado
[Qué debería pasar]

### Resultado actual
[Qué pasa realmente]

### Entorno
- Navegador / Versión:
- Dispositivo: Desktop / Móvil (modelo)
- URL:
- Fecha y hora:

### Captura de pantalla
[Si aplica]

### ¿Bloquea el lanzamiento?
- [ ] Sí
- [ ] No
```

---

## 🔍 Preguntas de Revisión

- [ ] ¿La plantilla pide suficiente información para reproducir el bug?
- [ ] ¿Diferenciamos entre bug crítico, mayor y menor?
- [ ] ¿Qué hacemos con bugs que no podemos reproducir?
- [ ] ¿Cómo priorizamos qué bugs arreglar primero?
- [ ] ¿Quién decide si un bug bloquea el lanzamiento?
