# Casos de Prueba — GuideFlow

## Formato

Cada caso incluye: nombre, pasos, resultado esperado, plataforma.

---

### CP-001: Registro de guía con email

**Pasos:** 1. Ir a /register → 2. Rellenar email + contraseña → 3. Submit
**Resultado:** Cuenta creada, redirect a onboarding paso 1
**Plataforma:** Desktop + Móvil

### CP-002: Registro con Google OAuth

**Pasos:** 1. Ir a /register → 2. Click "Continuar con Google" → 3. Autorizar
**Resultado:** Cuenta creada con datos de Google, redirect a onboarding
**Plataforma:** Desktop + Móvil

### CP-003: Onboarding completo

**Pasos:** 1. Datos básicos → 2. Conectar Stripe → 3. Crear actividad
**Resultado:** Microsite publicado, link copiable
**Plataforma:** Desktop + Móvil

### CP-004: Reserva y pago exitoso

**Pasos:** 1. Ir al microsite → 2. Elegir actividad → 3. Elegir fecha → 4. Pagar
**Resultado:** Booking creado, email enviado, plazas actualizadas
**Plataforma:** Móvil (principal)

### CP-005: Slot lleno → bloqueo

**Pasos:** 1. Llenar todas las plazas de un slot → 2. Intentar reservar otra
**Resultado:** Mensaje "No quedan plazas", sugerir otras fechas
**Plataforma:** Móvil

### CP-006: Cancelación con reembolso

**Pasos:** 1. Guía cancela booking → 2. El sistema procesa reembolso
**Resultado:** Status cambia, Stripe refund, email al cliente
**Plataforma:** Desktop

### CP-007: Pago fallido

**Pasos:** 1. Usar tarjeta de test que falla (`4000 0000 0000 0002`)
**Resultado:** Error claro al cliente, no se crea booking
**Plataforma:** Móvil

### CP-008: Fecha pasada

**Pasos:** 1. Intentar crear slot en fecha pasada
**Resultado:** Error de validación, no se crea
**Plataforma:** Desktop
