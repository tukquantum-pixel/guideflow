# Plan de Testing — GuideFlow

## ¿Qué probamos y cómo?

| Tipo | Herramienta | Qué cubre |
|------|------------|-----------|
| Unitarios | Vitest | Lógica de negocio (pricing, slots, cancelaciones) |
| Integración | Vitest + Prisma | API endpoints con BD real |
| E2E | Playwright | Flujo completo de reserva y pago |
| Manual | Guías reales | UX, onboarding, primera impresión |

## Flujo de pago sin dinero real

Stripe modo test: `sk_test_...` permite simular pagos completos con tarjetas de test (`4242 4242 4242 4242`). Webhooks de test se pueden triggear desde el CLI de Stripe.

## Casos de test críticos

| Caso | Prioridad |
|------|----------|
| Guía se registra y crea primera actividad | 🔴 Crítico |
| Cliente reserva y paga | 🔴 Crítico |
| Cliente cancela dentro de política → reembolso | 🔴 Crítico |
| Slot se llena → no se puede reservar más | 🟡 Alto |
| Webhook de Stripe falla → retry | 🟡 Alto |
| Guía sin Stripe intenta publicar actividad | 🟢 Medio |
| Página del guía carga en <2s en móvil | 🟢 Medio |

## Testing con guías reales (beta)

- **Cuándo:** Sprint 4 (semana 7)
- **Quién:** 3-5 guías invitados
- **Qué buscar:** Puntos de confusión en onboarding, bugs visuales en móvil
