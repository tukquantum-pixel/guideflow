## 📊 Objetivos de recuperación

| Métrica | Objetivo | Significado |
|---------|----------|-------------|
| RTO (Recovery Time Objective) | < 1 hora | Tiempo máx para restaurar servicio |
| RPO (Recovery Point Objective) | < 1 hora | Máx pérdida de datos aceptable |

## ⚙️ Plan de recuperación

| Escenario | Acción | Tiempo estimado |
|-----------|--------|-----------------|
| App crashea | Docker restart automático (`restart: unless-stopped`) | <30s |
| VPS caído | Levantar en nuevo VPS desde backup | <30min |
| BD corrupta | Restaurar desde último pg_dump o WAL | <1h |
| Ataque DDoS | Cloudflare modo "Under Attack" | <5min |
| Hack / breach | Rotación de secrets + restore desde backup | <2h |

## 🎯 Checklist de preparación

- [ ] Backups diarios verificados (restaurar 1x/mes)
- [ ] Secrets en vault (no en .env del servidor)
- [ ] Playbook documentado para cada escenario
- [ ] Contacto de emergencia del proveedor cloud
