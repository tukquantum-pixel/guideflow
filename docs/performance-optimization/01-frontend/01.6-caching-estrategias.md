## 📊 Niveles de caché en Next.js

| Nivel | Duración | Qué cachea | Config |
|-------|----------|-------------|--------|
| Browser cache | Headers directo | Assets estáticos (.js, .css, fonts) | `Cache-Control` |
| Next.js Data Cache | `unstable_cache` | Consultas Prisma frecuentes | TTL en función |
| ISR (Incremental Static) | `revalidate` | Páginas que cambian poco | `revalidate: 3600` |
| React `cache()` | Per-request | Deduplicar fetches en RSC | Automático |

## ⚙️ Patrón `unstable_cache` (usado en GuideFlow)

```typescript
import { unstable_cache } from "next/cache"

const getPopularRoutes = unstable_cache(
  async () => prisma.track.findMany({ where: { distance: { gte: 5 } }, take: 20 }),
  ["popular-routes"],
  { revalidate: 3600 } // 1 hora
)
```

## 🎯 Reglas de invalidación

- Crear/editar ruta → `revalidatePath("/explorar")`
- Editar perfil guía → `revalidateTag("guide-profile")`
- Nunca cachear datos de sesión/usuario autenticado
