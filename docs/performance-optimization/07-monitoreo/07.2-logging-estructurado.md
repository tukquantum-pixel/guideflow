## 🎯 Formato JSON para logs

```typescript
// lib/logger.ts
function log(level: string, message: string, meta?: Record<string, any>) {
  console.log(JSON.stringify({
    timestamp: new Date().toISOString(),
    level, message, ...meta,
    service: "guideflow", env: process.env.NODE_ENV,
  }))
}

export const logger = {
  info: (msg: string, meta?: any) => log("info", msg, meta),
  warn: (msg: string, meta?: any) => log("warn", msg, meta),
  error: (msg: string, meta?: any) => log("error", msg, meta),
}
```

## 📊 Qué loguear

| Evento | Nivel | Campos extra |
|--------|-------|-------------|
| Request recibido | info | method, path, ip |
| Query lenta (>100ms) | warn | query, duration_ms |
| Error 5xx | error | stack, request_id |
| Login exitoso | info | userId (hashed) |
| Pago completado | info | amount, plan |

## 🎯 Regla: NO loguear

- Passwords, tokens, API keys
- PII (email, teléfono) en texto plano
- Body completo de requests (solo en debug local)
