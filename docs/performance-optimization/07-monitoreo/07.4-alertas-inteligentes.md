## 📊 Matriz de alertas

| Alerta | Condición | Severidad | Canal | Acción |
|--------|-----------|-----------|-------|--------|
| API lenta | p95 > 500ms, 5min | Warning | Slack | Revisar queries |
| Error rate alto | 5xx > 5%, 2min | Critical | Telegram + Email | Investigar YA |
| BD saturada | connections > 80% | Warning | Slack | Revisar pool |
| Disco lleno | usage > 85% | Warning | Email | Limpiar logs/backups |
| SSL expira | <14 días | Info | Email | Renovar cert |
| Coste disparado | daily > 1.5x media | Warning | Slack | Revisar picos |
| App caída | Healthcheck fail 3x | Critical | Todo | Restart + investigar |

## ⚙️ Implementación

```yaml
# Uptime Robot (gratis, 50 monitors)
Monitor: https://guideflow.com/api/health
Interval: 5 min
Alert: Telegram + Email

# Grafana alerting
Alert: high_error_rate
Condition: avg(error_rate) > 5% for 2m
```

## 🎯 Regla anti-fatiga de alertas

- Máx 3 alertas/día en canal crítico
- Agrupar alertas del mismo tipo (debounce 5min)
- Severity clear: solo Critical despierta a alguien por la noche
