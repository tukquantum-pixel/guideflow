## 🎯 Qué es

Añadir más instancias del mismo servicio detrás de un load balancer. La app debe ser **stateless** para escalar horizontalmente.

## 📊 Requisitos para app stateless

| Requisito | Estado en GuideFlow | Acción |
|-----------|---------------------|--------|
| Sin archivos temporales en disco | ⚠️ Verificar | Subir a S3/R2 |
| Sesiones en BD/Redis, no en memoria | ✅ (NextAuth + BD) | OK |
| Caché compartida (Redis) | ⚠️ `unstable_cache` es local | Migrar a Redis para multi-instancia |
| Sin estado en memoria (globals) | ✅ | OK |
| Logs centralizados | ❌ | Loki/Grafana |

## ⚙️ Implementación con Docker Swarm

```bash
# Escalar a 3 instancias
docker service scale guideflow_app=3

# docker-compose con réplicas
deploy:
  replicas: 3
  update_config:
    parallelism: 1
    delay: 10s
```

## 🎯 Cuándo escalar horizontalmente

- Cuando 1 instancia no puede manejar el tráfico pico
- Cuando necesitas zero-downtime deployments
- A partir de ~5k-10k usuarios activos/día
