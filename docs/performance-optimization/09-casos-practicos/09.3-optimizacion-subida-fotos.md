## 🎯 Problema

Subir fotos durante una ruta. Sin optimización: imagen de 5MB pasa por el servidor Next.js → escribe en disco → sube a S3. Lento y caro.

## ⚙️ Solución: Upload directo a R2/S3

```typescript
// 1. API genera presigned URL
export async function POST(req: Request) {
  const { filename, contentType } = await req.json()
  const key = `photos/${Date.now()}_${filename}`
  const url = await generatePresignedPutUrl(key, contentType, 300) // 5min expiry
  return Response.json({ uploadUrl: url, publicUrl: `${CDN_URL}/${key}` })
}

// 2. Frontend sube DIRECTAMENTE a R2
const res = await fetch(uploadUrl, { method: "PUT", body: file, headers: { "Content-Type": file.type } })
```

## 📊 Impacto

| Métrica | Via servidor | Upload directo |
|---------|-------------|----------------|
| Tiempo (5MB) | 8-15s | 2-5s |
| CPU servidor | Alto (proxy) | 0% |
| Ancho de banda servidor | 5MB × 2 | 0MB |
| Coste | Alto | Solo storage R2 |

## 🎯 Compresión client-side

```typescript
// Antes de subir: resize a 2000px max y calidad 80%
import imageCompression from "browser-image-compression"
const compressed = await imageCompression(file, { maxWidthOrHeight: 2000, maxSizeMB: 1 })
```
