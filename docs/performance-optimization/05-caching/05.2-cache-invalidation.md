## 🎯 Patrones de invalidación

| Patrón | Cuándo | Ejemplo |
|--------|--------|---------|
| **TTL expiry** | Datos que cambian poco | Rutas populares (1h) |
| **Explicit delete** | Dato modificado | Guía edita perfil → `del guide:{id}` |
| **Tag-based** | Grupo de datos | `revalidateTag("routes")` |
| **Event-driven** | Webhook/mutation | Stripe webhook → invalidar suscripción |

## ⚙️ Implementación

```typescript
// Invalidación explícita
export async function updateGuide(id: string, data: any) {
  const guide = await prisma.guide.update({ where: { id }, data })
  // Invalidar todas las cachés relacionadas
  await redis.del(`guide:${id}`)
  await redis.del(`guide:${guide.slug}`)
  await redis.del("guides:featured")
  revalidateTag("guide-profile")
  return guide
}
```

## 📊 Anti-patterns

| ❌ No hacer | ✅ Hacer |
|-------------|---------|
| Cachear todo sin TTL | Siempre poner TTL (máx 24h) |
| Invalidar todo al cambiar algo | Invalidar solo las claves afectadas |
| Cache anidado (caché de caché) | Un solo nivel de caché por dato |
| Cachear datos de usuario actual | Solo cachear datos públicos/compartidos |
