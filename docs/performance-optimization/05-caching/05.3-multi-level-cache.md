## 📊 Niveles de caché en GuideFlow

```
[Cliente] → [CDN/Cloudflare] → [Next.js ISR] → [unstable_cache] → [Redis] → [PostgreSQL]
  L1 (browser)   L2 (edge)        L3 (server)     L4 (memory)      L5 (memory)   L6 (disk)
```

| Nivel | Latencia | Hit rate esperado | Coste |
|-------|----------|-------------------|-------|
| L1 Browser | 0ms | ~30% (assets) | 0€ |
| L2 CDN Edge | 5-20ms | ~40% (estáticos) | 0€ (Cloudflare gratis) |
| L3 ISR | 1-5ms | ~50% (páginas) | 0€ (incluido en Next.js) |
| L4 unstable_cache | 1-3ms | ~70% (datos) | 0€ |
| L5 Redis | 1-5ms | ~85% (hot data) | 0-5€/mes |
| L6 PostgreSQL | 5-50ms | 100% (source of truth) | Incluido |

## 🎯 Regla: la mayoría de requests no deben llegar a PostgreSQL

- **Objetivo**: >80% cache hit en L3-L5
- Solo mutaciones (POST/PUT/DELETE) llegan siempre a L6
