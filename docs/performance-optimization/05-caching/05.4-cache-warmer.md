## 🎯 Qué es

Job que pre-llena la caché antes de que los usuarios la necesiten. Evita "cold starts" después de deployments o reinicios.

## ⚙️ Implementación

```typescript
// scripts/cache-warmer.ts
import { getPopularRoutes, getFeaturedGuides } from "@/lib/data"

async function warmCache() {
  console.log("🔥 Warming cache...")
  await Promise.all([
    getPopularRoutes(), // Llena unstable_cache
    getFeaturedGuides(),
  ])
  console.log("✅ Cache warmed!")
}

warmCache().catch(console.error)
```

## 📊 Cuándo ejecutar

| Evento | Acción |
|--------|--------|
| Post-deploy | Ejecutar cache-warmer como post-build step |
| Cada 1h (cron) | Re-calentar datos expirados |
| Reinicio de Redis | Detectar y re-calentar automáticamente |

## 🎯 Prioridad de warming

1. Landing page (rutas populares, guías destacados)
2. Top 20 rutas por visitas
3. Top 10 perfiles de guías
