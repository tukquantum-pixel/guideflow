# 02.3 – Connection Pooling

| Metadata | Valor |

| Versión | 1.0 |
| Última actualización | 2026-03-08 |

| Estado | Borrador |

## 🎯 Problema

Cada request abre una conexión a PostgreSQL. Sin pooling, con 50 requests concurrentes se agotan las conexiones (max_connections = 100 por defecto).

## ⚙️ Solución: Prisma con connection pool

```typescript
// .env
DATABASE_URL="postgresql://user:pass@host:5432/guideflow?connection_limit=10&pool_timeout=10"
```

| Parámetro | Valor | Justificación |
|-----------|-------|---------------|
| `connection_limit` | 10 | Máx conexiones por instancia Prisma |
| `pool_timeout` | 10s | Tiempo máx esperando conexión |
| PostgreSQL `max_connections` | 100 | Default, suficiente hasta 10k usuarios |

## 📊 Con Prisma Accelerate (opcional)

```env
# Prisma Accelerate = connection pooler gestionado
DATABASE_URL="prisma://accelerate.prisma-data.net/?api_key=xxx"
DIRECT_URL="postgresql://user:pass@host:5432/guideflow"
```

## 🎯 Monitorizar

```sql
-- Verificar conexiones activas
SELECT count(*) FROM pg_stat_activity WHERE datname = 'guideflow';
```
