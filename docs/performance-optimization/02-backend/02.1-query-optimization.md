## 📐 Principios

1. **Seleccionar solo lo necesario** → usar `select` siempre
2. **Evitar N+1** → usar `include` con filtros
3. **Paginación cursor-based** → evitar `skip/take` grande

## ⚙️ Patrones optimizados

```typescript
// ✅ BIEN: solo campos necesarios
const guides = await prisma.guide.findMany({
  select: { id: true, name: true, slug: true, avatarUrl: true },
  where: { marketplaceEnabled: true },
  take: 20
})

// ❌ MAL: traer todo
const guides = await prisma.guide.findMany()

// ✅ Cursor pagination (O(1) vs O(n) del offset)
const next = await prisma.activity.findMany({
  cursor: { id: lastId }, skip: 1, take: 20,
  orderBy: { id: "asc" }
})

// ✅ Include con filtros (evita N+1)
const withBookings = await prisma.activity.findMany({
  include: {
    bookings: { where: { status: "CONFIRMED" }, take: 5 },
    guide: { select: { name: true } }
  },
  where: { active: true }, take: 20
})
```

## 📊 Excluir campos pesados

```typescript
// Excluir gpxData del listado (puede ser >100kb por registro)
const tracks = await prisma.track.findMany({
  select: { id: true, name: true, distance: true, elevationGain: true }
  // NO incluir gpxData ni geojson en listings
})
```
