## 📐 Principios

1. **Respuestas mínimas**: solo los campos que el cliente necesita
2. **Headers de cache**: `Cache-Control`, `ETag`, `Last-Modified`
3. **Compresión**: Gzip/Brotli automático
4. **HTTP status codes**: semánticos y correctos

## ⚙️ Headers de caché

```typescript
// API route con caché público
export async function GET() {
  const data = await getPopularRoutes()
  return Response.json(data, {
    headers: {
      "Cache-Control": "public, s-maxage=3600, stale-while-revalidate=86400"
    }
  })
}
```

## 📊 Patrón de respuesta estandarizado

```typescript
// Respuesta de éxito
{ data: [...], meta: { total: 100, page: 1, pageSize: 20 } }

// Respuesta de error
{ error: { code: "NOT_FOUND", message: "Ruta no encontrada" } }
```

## 🎯 Reglas de tamaño

| Endpoint | Max payload | Estrategia |
|----------|------------|------------|
| Listado rutas | <50kb | Paginación 20 items |
| Detalle ruta | <100kb | Sin GPX inline |
| GPX data | Variable | Endpoint separado, lazy |
| Búsqueda | <30kb | Solo campos de preview |
