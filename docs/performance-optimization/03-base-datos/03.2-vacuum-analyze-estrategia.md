# 03.2 – VACUUM & ANALYZE Estrategia

| Metadata | Valor |
|----------|-------|
| Versión | 1.0 |
| Última actualización | 2026-03-08 |
| Responsable | DBA |
| Estado | Borrador |

## 🎯 Qué hacen

- **VACUUM**: Libera espacio de filas borradas/actualizadas (dead tuples)
- **ANALYZE**: Actualiza estadísticas para el planificador de queries

## ⚙️ Autovacuum (activado por defecto)

```ini
# postgresql.conf
autovacuum = on
autovacuum_max_workers = 3
autovacuum_naptime = 60              # Check cada 60s
autovacuum_vacuum_threshold = 50     # Min dead tuples para VACUUM
autovacuum_analyze_threshold = 50    # Min cambios para ANALYZE
autovacuum_vacuum_scale_factor = 0.1 # VACUUM si >10% dead tuples
autovacuum_analyze_scale_factor = 0.05 # ANALYZE si >5% cambios
```

## 📊 Monitorizar dead tuples

```sql
SELECT relname, n_dead_tup, n_live_tup,
  round(n_dead_tup::numeric / GREATEST(n_live_tup, 1) * 100, 2) AS dead_pct,
  last_vacuum, last_autovacuum, last_analyze
FROM pg_stat_user_tables
ORDER BY n_dead_tup DESC LIMIT 10;
```
