## 📊 Estrategia de backups

| Tipo | Frecuencia | Herramienta | Impacto en rendimiento |
|------|-----------|-------------|----------------------|
| pg_dump lógico | Diario (03:00 UTC) | pg_dump | Medio (locks largos en tablas grandes) |
| WAL streaming | Continuo | pg_basebackup + WAL archiving | Bajo |
| Snapshot de disco | Diario | Cloud snapshot (Hetzner/DO) | Mínimo |

## ⚙️ pg_dump optimizado

```bash
# Ejecución paralela para reducir tiempo
pg_dump -Fd -j 4 -h localhost -U guideflow -d guideflow -f /backups/$(date +%Y%m%d)

# Compresión automática con formato custom
pg_dump -Fc -h localhost -U guideflow -d guideflow > /backups/guideflow_$(date +%Y%m%d).dump
```

## 📊 Restauración

```bash
# Restaurar desde dump
pg_restore -Fc -j 4 -d guideflow_restore /backups/guideflow_20260308.dump

# Tiempo estimado por tamaño
# 1GB → ~5 min | 10GB → ~30 min | 100GB → ~3h
```

## 🎯 Regla: PITR (Point-In-Time Recovery)

- WAL archiving activado para recovery a cualquier segundo
- `archive_mode = on` + `archive_command` configurado
- Retención: 7 días de WALs
