## ⚙️ EXPLAIN ANALYZE

```sql
EXPLAIN (ANALYZE, BUFFERS, FORMAT TEXT)
SELECT * FROM "Track" t
JOIN "Stage" s ON s."trackId" = t.id
WHERE t.distance > 10
ORDER BY t."elevationGain" DESC
LIMIT 20;
```

## 📊 Qué buscar en el plan

| Señal | Significado | Acción |
|-------|-------------|--------|
| `Seq Scan` | Escaneo secuencial completo | Añadir índice |
| `Index Scan` | Usa índice ✅ | OK |
| `Hash Join` | Join eficiente ✅ | OK |
| `Nested Loop` (con Seq Scan) | N+1 en SQL | Optimizar query |
| `Sort` (external) | No cabe en work_mem | Aumentar work_mem o añadir índice sorted |
| `Buffers: shared read` alto | Muchas lecturas de disco | Aumentar shared_buffers |

## 🎯 pg_stat_statements (top slow queries)

```sql
CREATE EXTENSION IF NOT EXISTS pg_stat_statements;

SELECT query, calls, mean_exec_time, total_exec_time
FROM pg_stat_statements
ORDER BY mean_exec_time DESC
LIMIT 10;
```

## 📏 Regla: toda query en producción debe ser < 100ms (p95)
