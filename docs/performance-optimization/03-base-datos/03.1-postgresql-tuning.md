# 03.1 – PostgreSQL Tuning

| Metadata | Valor |
|----------|-------|
| Versión | 1.0 |
| Última actualización | 2026-03-08 |
| Responsable | DBA / Infra |
| Estado | Borrador |

## 📊 Parámetros clave

| Parámetro | Default | Recomendado (4GB RAM VPS) | Impacto |
|-----------|---------|---------------------------|---------|
| `shared_buffers` | 128MB | 1GB (25% RAM) | Caché de datos en memoria |
| `effective_cache_size` | 4GB | 3GB (75% RAM) | Planificador de queries |
| `work_mem` | 4MB | 16MB | Memoria por operación (sort, hash) |
| `maintenance_work_mem` | 64MB | 256MB | VACUUM, CREATE INDEX |
| `random_page_cost` | 4 | 1.1 (con SSD) | Favorece Index Scan |
| `wal_buffers` | -1 (auto) | 64MB | Write-Ahead Log buffer |

## ⚙️ Aplicar en postgresql.conf

```ini
# /etc/postgresql/15/main/postgresql.conf
shared_buffers = 1GB
effective_cache_size = 3GB
work_mem = 16MB
maintenance_work_mem = 256MB
random_page_cost = 1.1
wal_buffers = 64MB
checkpoint_completion_target = 0.9
max_wal_size = 1GB
```

## 📏 Verificar rendimiento

```sql
-- Ver hit ratio del buffer cache (debe ser >99%)
SELECT sum(blks_hit) * 100 / sum(blks_hit + blks_read) AS cache_hit_ratio
FROM pg_stat_database WHERE datname = 'guideflow';
```
