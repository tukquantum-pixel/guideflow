# Stack técnico (resumen para no-técnicos)

## ¿Con qué herramientas lo construimos?

| Pieza | Qué hace | Tecnología | ¿Por qué esta? |
|-------|---------|-----------|----------------|
| **La web** | Lo que ven los guías y clientes | Next.js | Rápida, se posiciona bien en Google |
| **El estilo** | Que se vea bonita y profesional | Tailwind CSS | Rápido de diseñar |
| **La base de datos** | Guarda guías, actividades, reservas | PostgreSQL (Docker) | Robusta, gratuita, la conocemos bien |
| **Los pagos** | Cobra a los clientes y paga al guía | Stripe Connect | El líder mundial en pagos online |
| **Los emails** | Confirmaciones y recordatorios | Resend | Simple y fiable |
| **El servidor** | Donde vive la aplicación | VPS en Hetzner | 5€/mes, control total |
| **La máquina** | Empaqueta todo junto | Docker Compose | Todo funciona igual en local y en producción |

## ¿Cuánto cuesta mantener esto?

| Concepto | Coste mensual |
|----------|--------------|
| Servidor (Hetzner CPX21) | 5€ |
| Dominio | ~1€ |
| Emails (Resend) | 0€ (gratis hasta 100/día) |
| Stripe | 0€ (cobra al cliente directamente) |
| **Total** | **~6€/mes** |

Con 1.000 guías activos, subiríamos el servidor a ~15€/mes. Escalable y barato.

## ¿El equipo tiene experiencia con esto?

Sí. Hemos usado este mismo stack (Next.js + PostgreSQL + Docker + Stripe) en otros proyectos del workspace. No hay nada experimental.
