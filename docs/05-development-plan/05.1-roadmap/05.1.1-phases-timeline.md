# Roadmap y Sprints — GuideFlow

## Sprint 1 — Fundaciones (Semana 1-2)

- [ ] Inicializar Next.js 15 (App Router)
- [ ] Configurar Docker Compose (PostgreSQL 16)
- [ ] Configurar Prisma con schema base
- [ ] Auth con Auth.js (email + Google)
- [ ] Layout base y sistema de diseño (Tailwind)
- [ ] Onboarding Stripe Connect Express

## Sprint 2 — Core (Semana 3-4)

- [ ] CRUD de actividades
- [ ] Upload de fotos (almacenamiento local/S3)
- [ ] Motor de disponibilidad (slots)
- [ ] Microsite público del guía (SSR)

## Sprint 3 — Pagos y Reservas (Semana 5-6)

- [ ] Checkout con Stripe
- [ ] Webhooks (pago, reembolso)
- [ ] Dashboard de reservas del guía
- [ ] Emails transaccionales (Resend)

## Sprint 4 — Polish y Lanzamiento (Semana 7-8)

- [ ] Recordatorios automáticos (24h antes)
- [ ] Cancelaciones con política configurable
- [ ] QR code del microsite
- [ ] Landing de captación de guías
- [ ] Tests E2E del flujo completo
- [ ] Deploy a producción (VPS + Caddy)

## Margen para imprevistos

Cada sprint tiene 2 semanas pero ~3 días de buffer (sábados). Si un sprint se retrasa, el buffer absorbe antes de mover los siguientes.

---

## 🔍 Preguntas de Revisión

- [ ] ¿Qué pasa si una fase se retrasa? ¿Cómo ajustamos las siguientes?
- [ ] ¿Hay dependencias críticas entre fases? (ej: no podemos hacer frontend sin API)
- [ ] ¿Quién decide cuándo una fase está completada?
- [ ] ¿Hemos dejado margen para imprevistos? (enfermedad, bugs)
- [ ] ¿Cuándo mostramos el producto a usuarios reales por primera vez?

## 🎯 Preguntas de Mejora Continua

- [ ] ¿Cómo recogemos feedback de seguridad de los guías?
- [ ] ¿Qué hacemos cuando descubrimos una vulnerabilidad?
- [ ] ¿Cómo priorizamos deuda técnica vs nuevas features?
- [ ] ¿Cuándo fue la última vez que actualizamos dependencias?
- [ ] ¿El equipo tiene tiempo para aprender y mejorar?
- [ ] ¿Cómo medimos la calidad del código? (coverage, complejidad)
- [ ] ¿Qué nos impide tener un sistema "battle-tested"?
