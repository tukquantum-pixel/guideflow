# Flujo de Pago — GuideFlow

## Lo que ve el cliente al pagar

1. Resumen: "Barranquismo en Ordesa — Sáb 15 Mar 9:00 — 2 personas — 120€"
2. Formulario de tarjeta (Stripe Checkout)
3. Pantalla de éxito: "✅ ¡Reserva confirmada! Te hemos enviado los detalles por email."

## Lo que recibe el guía

- Notificación por email: "Nueva reserva de Juan García para el sábado 15"
- Dashboard actualizado con el nuevo booking
- El dinero llega a su cuenta automáticamente (1-2 días hábiles)

## El flujo del dinero

```
Cliente paga 120€ (2 personas × 60€)
  → Stripe cobra ~1.90€ (fee del banco)
  → Si plan Starter: GuideFlow cobra 6€ (5%)
  → Guía recibe: 112.10€ (Starter) o 118.10€ (Pro)
```

## Reembolsos: quién decide

| Escenario | Quién decide | Acción |
|-----------|-------------|--------|
| Cliente cancela dentro de política | Automático | Reembolso según política del guía |
| Cliente cancela fuera de política | El guía | Puede decidir reembolsar o no |
| Guía cancela la actividad | Automático | Reembolso total al cliente |
| Pago marcado como fraude | Stripe | Disputa automática, se retiene el dinero |

## Webhooks que escuchamos

| Evento Stripe | Acción |
|--------------|--------|
| `checkout.session.completed` | Crear booking, enviar confirmación |
| `payment_intent.succeeded` | Marcar como pagado |
| `charge.refunded` | Actualizar estado, notificar |
| `account.updated` | Actualizar KYC del guía |
