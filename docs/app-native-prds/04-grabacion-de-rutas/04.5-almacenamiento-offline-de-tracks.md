# PRD: Almacenamiento Offline de Tracks

| Metadata | Valor |
|----------|-------|
| **Versión** | 1.0 |
| **Fecha** | 2026-03-10 |
| **Estado** | Borrador |

## 🎯 Objetivo

Guardar tracks grabados localmente (MMKV) cuando no hay conexión, y sincronizar al recuperar red.

## Storage

```typescript
// MMKV storage for tracks
const trackStorage = new MMKV({ id: 'tracks' })

interface PendingTrack {
  id: string           // UUID local
  points: TrackPoint[]
  startedAt: number
  finishedAt: number
  distance: number
  elevationGain: number
  synced: boolean
}
```

## Flujo offline

1. Grabación termina → guardar en MMKV con `synced: false`
2. Cada 30s durante grabación → snapshot parcial en MMKV (crash recovery)
3. Al recuperar conexión → NetInfo listener dispara sync
4. POST `/api/tracks` → si 200, marcar `synced: true`
5. Tracks sincronizados se borran de MMKV tras 7 días

## Cola de sincronización

```typescript
interface SyncQueue {
  pendingTracks: string[]     // IDs de tracks por subir
  retryCount: Record<string, number>
  lastAttempt: Record<string, number>
}
```

## 🧪 Tests

| Escenario | Esperado |
|-----------|----------|
| Grabar sin conexión | Track se guarda en MMKV |
| Recuperar conexión | Track se sube automáticamente |
| Fallo de subida | Reintento exponencial (1m, 5m, 15m) |
| App crashea mid-recording | Snapshot parcial se recupera al reabrir |

3 días
