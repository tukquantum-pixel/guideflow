## 🎯 Descripción funcional

Los usuarios pueden reportar el estado actual de la ruta: sendero cortado, árbol caído, nieve, barro, agua helada, señalización dañada. Los reportes se muestran a futuros senderistas con indicación de antigüedad y fiabilidad.

## 🧠 Problema que resuelve

- "El sendero estaba cortado por un derrumbe y no lo sabía"
- Información actualizada por quienes realmente están en el terreno
- Red de inteligencia colaborativa sobre estado de senderos

## 👤 Experiencia de usuario

1. Durante la ruta, botón "⚠️ Reportar condición"
2. Seleccionar tipo: sendero cortado, barro, nieve, peligro, otro
3. Opcionalmente adjuntar foto y nota
4. El reporte aparece inmediatamente para otros usuarios
5. Los usuarios pueden confirmar/desmentir reportes previos

## ⚙️ Requisitos técnicos

- [ ] Modelo `ConditionReport` (userId, trackId, stageId, type, note, photoUrl, lat, lng, createdAt)
- [ ] API CRUD `/api/community/conditions`
- [ ] Sistema de confirmación/votos para validar reportes
- [ ] Expiración automática de reportes (14 días sin confirmación)
- [ ] Notificación a usuarios con la ruta guardada

## 📊 Criterios de éxito

- >20 reportes por semana en rutas populares
- Tasa de confirmación >60%
- Tiempo desde reporte hasta visibilidad: <5 minutos

## 🔗 Dependencias

- GPS activo para geolocalización del reporte
- Comunidad activa de usuarios
- Disponible para todos los planes (incentivo de uso)
