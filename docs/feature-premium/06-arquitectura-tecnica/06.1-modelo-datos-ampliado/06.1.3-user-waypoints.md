| Metadata | Valor |
|----------|-------|

| Última actualización | 2026-03-08 |
| Responsable | Backend |
| Estado | Borrador |

## 📊 Modelo Prisma propuesto

```prisma
model UserWaypoint {
  id        String   @id @default(uuid())
  userId    String
  trackId   String?  // Opcional: asociado a una ruta
  type      String   // "water", "viewpoint", "danger", "shelter", "camp", "other"
  name      String
  note      String?  @db.Text
  lat       Float
  lng       Float
  elevation Float?
  photoUrl  String?
  isPublic  Boolean  @default(false)
  approved  Boolean  @default(false) // Para waypoints públicos
  createdAt DateTime @default(now())

  user  AppUser @relation(fields: [userId], references: [id], onDelete: Cascade)
  track Track?  @relation(fields: [trackId], references: [id])

  @@index([userId])
  @@index([trackId, isPublic])
  @@index([lat, lng])
}
```

## 🎯 Propósito

Waypoints creados por el usuario durante rutas. Privados por defecto, con opción de publicar para la comunidad (requiere moderación).

## ⚙️ Consideraciones

- Sincronización offline: crear localmente con UUID, sync al tener conexión
- Moderación de waypoints públicos antes de visibilidad
- Integración con sistema de Checkpoints existente (los públicos aprobados podrían convertirse en Checkpoints)
