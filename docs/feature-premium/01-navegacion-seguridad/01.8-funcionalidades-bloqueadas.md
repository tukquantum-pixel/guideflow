## 🔴 BLOQUEADO — No se puede hacer AHORA (5/12)

### 1. Mapas 3D interactivos

| Campo | Detalle |
|-------|---------|
| **Qué es** | Visualizar el terreno en 3D con la ruta superpuesta |
| **Por qué NO se puede ahora** | Leaflet es 2D. Requiere **MapLibre GL JS** con `map.setTerrain()` + tiles DEM (Digital Elevation Model) |
| **Qué necesitamos** | (1) Migrar a MapLibre GL JS, (2) Servidor de tiles DEM o usar fuente gratuita (Terrain RGB de AWS), (3) Optimizar para móviles (WebGL) |
| **Esfuerzo estimado** | 2-3 sprints |
| **Spec**: | `docs/feature-premium/01-navegacion-seguridad/01.7-maplibre-especificacion.md` |

### 2. Animación de ruta (fly-through)

| Campo | Detalle |
|-------|---------|
| **Qué es** | Reproducir un "vuelo virtual" a lo largo de la ruta en 3D |
| **Por qué NO se puede ahora** | Depende del 3D (punto anterior). Requiere MapLibre GL + `map.flyTo()` animado a lo largo del track |
| **Qué necesitamos** | (1) Implementar el punto 1 primero, (2) Programar interpolación suave de cámara entre puntos del track |
| **Esfuerzo estimado** | 1 sprint (después del 3D) |

### 3. Mapas offline

| Campo | Detalle |
|-------|---------|
| **Qué es** | Descargar mapas completos para usar sin conexión en montaña |
| **Por qué NO se puede ahora** | Es una funcionalidad **100% móvil**. Los navegadores web no permiten almacenar tiles offline de forma fiable. Requiere **app nativa** (React Native/Capacitor) con almacenamiento local |
| **Qué necesitamos** | (1) App nativa con React Native o Capacitor, (2) Generar paquetes PMTiles/MBTiles por zona geográfica, (3) Download manager con progreso, (4) Cache inteligente de tiles |
| **Esfuerzo estimado** | 3-4 sprints + app nativa |
| **Spec**: | `docs/feature-premium/01-navegacion-seguridad/01.2-mapas-offline.md` |

### 4. Alertas de desvío

| Campo | Detalle |
|-------|---------|
| **Qué es** | Avisar al usuario si se desvía más de X metros del sendero |
| **Por qué NO se puede ahora** | Requiere **GPS en tiempo real** del dispositivo + comparación continua con el track. Los navegadores web no proporcionan GPS de alta frecuencia ni geofencing en segundo plano |
| **Qué necesitamos** | (1) App nativa con acceso a GPS, (2) API Geolocation de alta frecuencia, (3) Cálculo `pointToLineDistance` con turf.js, (4) Notificaciones push/sonido |
| **Esfuerzo estimado** | 2-3 sprints + app nativa |
| **Spec**: | `docs/feature-premium/01-navegacion-seguridad/01.1-alertas-desvio.md` |

### 5. Live tracking (seguimiento en vivo)

| Campo | Detalle |
|-------|---------|
| **Qué es** | Que un familiar/amigo vea en tiempo real dónde estás en la ruta |
| **Por qué NO se puede ahora** | Requiere: (1) GPS en tiempo real (app nativa), (2) **WebSocket server** para transmitir posiciones, (3) Gestión de sesiones de tracking, (4) Frontend de visualización para el "espectador" |
| **Qué necesitamos** | (1) App nativa, (2) WebSocket server (Socket.io o WS), (3) API de sesiones de tracking, (4) Panel de seguimiento web para familiares |
| **Esfuerzo estimado** | 4-5 sprints |
| **Spec**: | `docs/feature-premium/01-navegacion-seguridad/01.3-live-tracking.md` |
