## 🎯 Descripción funcional

Enriquecimiento de los Checkpoints existentes con información detallada: horarios de refugios, caudal de fuentes, estado de puentes, cobertura móvil, disponibilidad de servicios. Incluye alertas proactivas basadas en la posición ("Fuente a 500m a la izquierda").

## 🧠 Problema que resuelve

- "No sabía que había una fuente a 500m del camino"
- "El refugio estaba cerrado y no lo sabía"
- Información contextual que enriquece la experiencia

## 👤 Experiencia de usuario

1. En navegación activa, notificación "Fuente a 500m → desvío izquierda"
2. En mapa, POIs con iconos ricos y tooltip con detalles
3. Filtros de POIs: mostrar solo fuentes, solo refugios, etc.
4. Cada POI muestra última actualización y fiabilidad

## ⚙️ Requisitos técnicos

- [ ] Ampliar modelo Checkpoint con campos: `openingHours`, `phoneNumber`, `mobileSignal`, `lastVerified`
- [ ] Sistema de alertas proactivas por proximidad (geofencing)
- [ ] Enriquecimiento de datos desde OSM (Overpass API tags)
- [ ] Interfaz de actualización comunitaria (reportar estado)

## 📊 Criterios de éxito

- >80% de checkpoints con información ampliada
- Alertas de proximidad con precisión ±50m
- Datos actualizados cada temporada

## 🔗 Dependencias

- Modelo Checkpoint existente
- GPS activo (para alertas de proximidad)
- Plan Explorer o superior
