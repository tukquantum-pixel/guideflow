## 🎯 Descripción funcional

Mapa de calor (heatmap) que muestra las zonas más y menos concurridas de cada ruta, basado en datos anónimos de GPS de usuarios de GuideFlow. Permite evitar aglomeraciones eligiendo horarios o rutas alternativas.

## 🧠 Problema que resuelve

- "Fui al Montseny un domingo y estaba lleno de gente"
- Encontrar momentos de tranquilidad en rutas populares
- Distribuir el tráfico para reducir impacto ambiental

## 👤 Experiencia de usuario

1. En la ficha de ruta, pestaña "Tráfico" con mapa de calor
2. Selector de día/hora para ver pronóstico de afluencia
3. Indicadores: 🟢 Poco tráfico / 🟡 Moderado / 🔴 Muy concurrido
4. Recomendación de mejor horario para ir

## ⚙️ Requisitos técnicos

- [ ] Modelo `TrafficData` (lat, lng, timestamp, anonymousUserId)
- [ ] Recolección anónima de posiciones GPS (opt-in)
- [ ] Generador de heatmap con agregación por cuadrículas H3
- [ ] API `GET /api/premium/traffic?routeId=&day=&hour=`
- [ ] Procesamiento batch nocturno para actualizar pronósticos

## 📊 Criterios de éxito

- Mínimo 100 data points por ruta popular para predicción fiable
- Precisión de pronóstico >70% (validado con datos históricos)

## 🔗 Dependencias

- Base instalada de usuarios con GPS activo
- Consentimiento de compartir ubicación anónima (RGPD)
- Plan Peak
