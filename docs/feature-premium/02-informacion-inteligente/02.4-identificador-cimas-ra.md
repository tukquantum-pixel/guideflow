## 🎯 Descripción funcional

El usuario apunta con la cámara del móvil hacia las montañas y la app superpone el nombre, altitud y distancia de cada pico visible. Utiliza GPS + brújula + base de datos de picos para calcular qué cimas son visibles desde la posición actual.

## 🧠 Problema que resuelve

- "¿Cómo se llama esa montaña?"
- Aprendizaje geográfico y cultural del entorno
- Feature wow/viral para compartir en redes sociales

## 👤 Experiencia de usuario

1. Pulsar "Identificar cimas" → abre la cámara
2. Apuntar hacia el horizonte montañoso
3. Etiquetas flotantes con nombre, altitud y distancia
4. Tap en una etiqueta → ficha del pico (foto, rutas que pasan, historia)

## ⚙️ Requisitos técnicos

- [ ] Base de datos de picos: `Peak` (name, lat, lng, elevation, region)
- [ ] Algoritmo de visibilidad: línea de vista desde posición + altitud usuario
- [ ] Sensores: GPS + magnetómetro + acelerómetro
- [ ] Librería AR: ARKit (iOS) / ARCore (Android) vía React Native
- [ ] API `GET /api/peaks/identify?lat=&lng=&heading=&pitch=`

## 📊 Criterios de éxito

- Identificación correcta >90% de picos visibles
- Respuesta en <2 segundos
- BD de picos: >2000 cimas en España

## 🔗 Dependencias

- Base de datos de picos (importar desde IGN, OpenStreetMap)
- Hardware: giroscopio + brújula en el dispositivo
- Plan Peak
