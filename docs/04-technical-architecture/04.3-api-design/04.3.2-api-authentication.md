# Autenticación API — GuideFlow

## Estrategia

- **Auth.js (NextAuth)** con sesiones JWT almacenadas en cookie httpOnly
- **Providers:** Email/Password + Google OAuth
- **Sesiones:** PostgreSQL como adapter (tabla `sessions`, `accounts`, `users`)

## Flujo

```
1. Guía hace login (email/Google)
2. Auth.js crea JWT y lo almacena en cookie httpOnly
3. Cada request del frontend envía la cookie automáticamente
4. Middleware de Next.js valida la sesión
5. Si expira, redirect a login
```

## Protección de rutas

| Tipo de ruta | Protección |
|-------------|-----------|
| `/api/guide/*` | JWT obligatorio, RLS por `guide_id` |
| `/api/activities/*` | JWT + verificar que la actividad pertenece al guía |
| `/api/public/*` | Sin auth (público) |
| `/api/webhooks/stripe` | Verificación `stripe-signature` (no JWT) |

## Permisos

Un guía **solo puede ver y editar sus propios datos**. Esto se refuerza a dos niveles:

1. **Middleware:** Extrae `guide_id` del JWT
2. **Query Prisma:** Filtra siempre con `WHERE guide_id = $guideId`

## Rate limiting

- Rutas públicas: 60 requests/min por IP
- Rutas protegidas: 120 requests/min por usuario
