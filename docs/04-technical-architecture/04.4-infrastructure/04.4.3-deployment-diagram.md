# Diagrama de Despliegue — GuideFlow

## Arquitectura

```
VPS Hetzner (CPX21 - 5€/mes)
├── Docker Compose
│   ├── app (Next.js · puerto 3000)
│   ├── db (PostgreSQL 16 · puerto 5432 solo interno)
│   └── caddy (reverse proxy + SSL automático)
├── Backups
│   └── pg_dump diario → Object Storage Hetzner
└── Monitorización
    └── Healthcheck HTTP cada 5 min
```

## Single points of failure

| SPOF | Riesgo | Mitigación |
|------|--------|------------|
| VPS se cae | Medio | Hetzner tiene SLA 99.9%, backups diarios |
| PostgreSQL se corrompe | Bajo | pg_dump diario a object storage separado |
| Stripe down | Bajo | Stripe tiene SLA 99.99% |

## Plan de escalado

| Fase | Guías | Infra |
|------|-------|-------|
| MVP (0-100) | CPX21 (5€/mes) | Todo en un VPS |
| Crecimiento (100-500) | CPX31 (15€/mes) | VPS más grande |
| Escala (500+) | 2 VPS + LB | App separada de DB |

## 🏢 Preguntas de Escalabilidad Enterprise

- [ ] ¿Nuestra arquitectura soporta alta disponibilidad? (múltiples instancias, balanceo)
- [ ] ¿Qué pasa si un servidor se cae? ¿Hay failover automático?
- [ ] ¿Cómo escalamos horizontalmente cuando tengamos 10.000 guías activos?
- [ ] ¿La base de datos soporta replicación? (lectura/escritura separadas)
- [ ] ¿Usamos CDN para contenido estático? (imágenes de los guías)
- [ ] ¿Cómo medimos y garantizamos tiempos de respuesta <200ms?
- [ ] ¿Hay límites de uso documentados? (API calls, reservas, etc.)

## 📜 Controles Técnicos ISO 27001

- [ ] ¿Tenemos sistemas de detección de intrusiones?
- [ ] ¿Los logs están centralizados y protegidos contra manipulación?
- [ ] ¿Cómo detectamos accesos anómalos? (ej: un guía accede desde 3 países en 1h)
- [ ] ¿Las comunicaciones internas entre servicios están cifradas? (mTLS)
- [ ] ¿Los contenedores se ejecutan con privilegios mínimos? (no root)
- [ ] ¿Las imágenes de Docker se escanean en busca de vulnerabilidades?
- [ ] ¿Cómo gestionamos parches de seguridad en el SO del VPS?
- [ ] ¿Hay segmentación de red? (base de datos no accesible desde internet)
