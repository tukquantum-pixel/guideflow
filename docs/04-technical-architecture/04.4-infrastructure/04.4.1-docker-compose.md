# Docker Compose + Infraestructura — GuideFlow

## docker-compose.yml

```yaml
version: '3.8'

services:
  app:
    build: .
    restart: unless-stopped
    ports:
      - "3000:3000"
    env_file:
      - .env
    depends_on:
      - db

  db:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_USER: guideflow
      POSTGRES_PASSWORD: ${DB_PASSWORD}
      POSTGRES_DB: guideflow
    volumes:
      - pgdata:/var/lib/postgresql/data
    ports:
      - "5432:5432"

volumes:
  pgdata:
```

## Variables de entorno

| Variable | Secreta | Dev | Prod |
|----------|---------|-----|------|
| `DATABASE_URL` | ✅ | `postgresql://guideflow:dev@localhost:5432/guideflow` | Diferente |
| `STRIPE_SECRET_KEY` | ✅ | `sk_test_...` | `sk_live_...` |
| `STRIPE_WEBHOOK_SECRET` | ✅ | `whsec_...` | Diferente |
| `NEXTAUTH_SECRET` | ✅ | Random string | Diferente |
| `NEXTAUTH_URL` | ❌ | `http://localhost:3000` | `https://app.guideflow.com` |
| `NEXT_PUBLIC_STRIPE_KEY` | ❌ | `pk_test_...` | `pk_live_...` |
| `RESEND_API_KEY` | ✅ | `re_test_...` | `re_...` |

## Si falta una variable obligatoria

La app **NO arranca** y muestra un error claro:

```
ERROR: Missing required env: STRIPE_SECRET_KEY
```
